Course Description


This class covers principles and practices of modern web application design, development and deployment. This class focuses on web server code development, deployment, and management, and will cover the aspects of front-end development needed to produce functional websites. Students will explore various types of web site hosting options. Students will use web application frameworks, databases, developer tools (such as testing suites and source control tools) and other resources to build and deploy web applications. Students will be introduced to security issues, and user experience considerations. Must be taken A-F.


Course Objectives 


At the end of this class, students will be able to use the following tools and technologies to develop, deploy and manage a website. Students will understand the role of each of these tools/components, and their purpose in modern application development and deployment.


• HTML

• CSS

• JSON

• JavaScript

• Vue.js

• JSON

• HTTP

• NPM

• Node.js

• Express

• Relational databases such as SQLite or PostgreSQL

• ORM

• RESTful APIs

• Heroku

• Deployment tools

• Git/GitHub Version control software and code hosting site


Students will also consider user interface design (UX), accessibility, and web application security.

Suggested Time Requirements


A six-credit class required 6 hours of classroom time. We’ve been assigned 4 hours per week, so you should prepare to spend an extra 2 hours working on classroom topics. 


In addition, a typical college class expects students to spend roughly 2 times the class hours per week on coding, assignments, projects, reading, and other class work. So, you should expect to spend a minimum of an additional 12 hours per week working on this class.


This class might be harder than you expect. This class expects a lot of work! That's because learning to program is challenging, and takes time and practice. You will get out of this class what you put in. The more you practice, the more you will understand, and you will become a better programmer.  The better programmer you are, the better the chance of finding employment in the field.


Every student is different and you may require more time, or less time, to be successful in this class.


Software development employers are not interested in software development students whose priority for their classes is simply the final grade. Instead, make it a priority to understand the technologies used; so that you can apply them to projects beyond your assignments. Employers express a strong preference for students who spend the time to understand the technologies they are working with; for students who work on programs and projects beyond the required class work; for students who are resourceful and show initiative in solving problems, who ask questions; and demonstrate enthusiasm for software development. This approach builds your programming skills, understanding, and employability. And, this approach usually has a strong positive effect on your grades!

Cheating and Plagiarism


Cheating, plagiarism, and/or academic dishonesty will not be tolerated in this class. 


Plagiarism includes, but is not limited to, copying, adapting, or paraphrasing of any part of any published or unpublished work of another person without full and clear acknowledgment. 


Plagiarism also includes the unacknowledged use of materials prepared by another person, for example, from a web forum, another student, tutor, or any other person.


You are encouraged to work together with other students, to form study groups, and provide academic help and support to each other. DO NOT give other students copies of your work, DO NOT write code for other students, and DO NOT let others copy your work. 


Always submit your own work, that you have created yourself, for assignments, labs, and projects. You are encouraged to research online and review other programs, tutorials, videos, and code samples, but DO NOT copy other code. If you use or adapt any part of any other program or tutorial, you must identify this clearly in your code.


You MUST cite all sources in all programs, projects, labs and assignments. For example, if you have used OR adapted any code from a textbook, tutorial, video, or website you MUST attribute the source and identify all of the code that was obtained or adapted from this source. Adding comments to your code identifying the affected code to identifying your sources(s) by URL, or by book title plus page number, is acceptable.


If you work on an assignment, or lab, or project with another student, a tutor, or any other person, you MUST use comments in your code to identify the relevant code in your work. Include the name of the other person(s) who assisted you with your code. 


If you are not sure whether your work requires citations, or if your work may be considered plagiarized, or if you need assistance creating appropriate citations, then please contact me for guidance BEFORE you submit it for grading.


Student work may be checked at any time by automatic plagiarism detection tools. 


Any assignment, project, lab, or other work which is partly or wholly plagiarized, or violates any of the above policies, or is in violation of the Student Code of Conduct in any way, will receive a 0 grade. Students will NOT be permitted to repeat or make up such work. 


All plagiarism incidents will be reported to the college’s student conduct authorities for investigation. Consequences may include academic suspension or expulsion from the college.


For more information, please read the Student Code of Conduct or contact me. http://www.minneapolis.edu/About-Us/Policies/Student-Affairs-Policies/404-Student-Code-of-Conduct/40401-Student-Code-of-Conduct-Procedures

Grading and Assessment


Classroom Labs


The class material is divided into topics. Each topic has an associated lab. Labs will include programming programs to analyze and solve, plus short-answer questions on concepts. 


You will typically start the lab in class. You will finish any remaining work outside of class. You should submit the rest of your lab work to the correct dropbox by the deadline given – this typically is the day before the next class session. 


Some labs ask you to build your own version of an application. The code for the finished app will be provided for you but you should NOT simply copy and paste the code. Even though the final code is available to you, it should be used as a reference only. You need to build the applications yourself so you can go through the process of configuring, creating, troubleshooting, and adding features to an application. The code samples provided have hidden bytes which will be detected if you copy and paste the example code. Code that is copied from the sample code will be given a 0 grade. 


Attendance Quizzes


Attendance is expected in every class session. You must take the weekly attendance quiz in class for credit for attendance. You may miss up to two class sessions without affecting your attendance grade.


Reading and Review Quizzes


You should prepare for each class by reviewing the materials in advance of that class. You will take a short quiz with a few multiple-choice questions. This quiz also asks what the most confusing part of last week’s class was; and what questions you have on the upcoming week’s materials.


Midterm Project


You will develop a client-side application, using HTML, CSS and JavaScript.

Details for the project will be given in D2L. 


Final Project


Your final project will ask you to build a larger, more complex application with both client and server side. This will be an individual project. You will need to use all the skills learned in class. As part of the project, you will participate in a code review, and present your final project to the class during the final week of the semester.

Full details for the final project will be given in D2L.
